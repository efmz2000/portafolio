-Pero de una manera mas a fin a la carre se habla de tres puntos para definir un paradigma:
	1.Es un estilo fundamental de programación.
	2.Es una forma de construir estructuras y elementos de programación.
	3.Es la forma en que el lenguaje ve al mundo.

-Paradigma Orientado a Objetos.
	ANALIZA, DISEÑA y CONSTRUYE programas a  partir de objetos
	*Se debe de IDENTIFICAR, CARACTERIZAR  Y ESTABLECER LAS ERLACIONES DE OBJETOS
 
-Se hablo sobre algunos conceptos del Paradigma Orientado a Objetos como: 
	*BASTRACCIÓN -> siendo esta una REPRESENTACION no exacta de la realidad. 
	Omitiendo detalles y enfocandose en lo importante.
	
	*Encapsulamiento -> Se trata de tomar acomodar los datos y los métodos de una estructura 
	para evitar que otro metodo ajeno a este objeto modifique esta informacion.1
	
	*Clases -> La clase es un molde para luego crear objetos en base a estas. Definiendo
	los atributos que deben tener los objetos y los metodos que se prodran realizar

	*Atributos -> Los atributos son caracteristicas especificas de cada objeto para poder 
	diferenciarlo

	*Métodos -> Los metodos son instrucciones que se le pueden dar  a los objetos para 
	modificar o revisar sus datos 

	*Instancias -> las instancias es una represenentacion de un objeto creado a partir de 
	una clase

	*Herencia -> La herencia es el uso de los atributos de una clase pre-existente para la 
	creación de una nueva clase. Este metodo da la oportunidad de tener caracteristicas y
	metodos de la clase anterior como crear nuevas para esta nueva 

	**Polimorfismo -> el polimorfismo es la capacidad de los objetos de poder usar una funcion 
	ya establecida en otra clase dandoles otros parametros
